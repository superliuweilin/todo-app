# 大模型（Large Language Model, LLM）教学资料

---

## 一、大模型基本概念

### 1.1 什么是大模型

**大模型**（Large Language Model，简称 LLM）是指参数量巨大（通常数十亿到数千亿）的深度学习模型，通过对海量文本数据的预训练，学习语言的统计规律和知识表示，从而具备强大的自然语言理解和生成能力。

通俗理解：大模型就像是一个"读过全世界几乎所有书籍和网页"的学生，它虽然不一定记住了每句话，但学会了语言的模式、推理的方式和各种领域的知识，因此能回答问题、写文章、翻译、编程等。

### 1.2 核心术语

| 术语 | 解释 |
|------|------|
| **Token（词元）** | 文本被切分后的最小处理单元。一个中文汉字约 1-2 个 token，一个英文单词约 1-3 个 token。模型按 token 计费和限制上下文长度。 |
| **参数（Parameter）** | 模型中可学习的权重数量，通常在数十亿（B）到数千亿级别。参数越多，模型的"记忆容量"和表达能力越强，但计算成本也越高。 |
| **上下文窗口（Context Window）** | 模型一次能处理的 token 数量上限。早期 GPT-2 只有 1024 tokens，现在主流模型可达 128K 甚至 1M tokens。 |
| **预训练（Pre-training）** | 在海量无标注文本上，通过"预测下一个词"（Next Token Prediction）等自监督任务训练模型，使其学会语言模式和知识。 |
| **微调（Fine-tuning）** | 在预训练模型基础上，用特定领域的有标注数据进一步训练，使模型适应特定任务。 |
| **涌现能力（Emergent Ability）** | 当模型规模超过某个阈值后，突然"涌现"出的预训练时未专门训练过的能力，如逻辑推理、代码生成、思维链等。 |
| **幻觉（Hallucination）** | 模型生成的内容看似合理但与事实不符，是当前大模型的主要问题之一。 |
| **提示工程（Prompt Engineering）** | 设计输入文本的策略和技巧，以引导模型生成更准确、更有用的输出。 |

### 1.3 大模型的能力

- **自然语言理解**：文本分类、情感分析、实体识别、阅读理解
- **自然语言生成**：文章写作、摘要生成、对话、翻译
- **知识问答**：回答各领域知识性问题
- **代码生成**：根据自然语言描述生成代码，调试、解释代码
- **推理能力**：数学推理、逻辑推理、常识推理
- **多模态**（部分模型）：理解图像、音频、视频等多种输入

## 二、基础理论

### 2.1 Transformer 架构

现代大模型几乎全部基于 **Transformer** 架构（Vaswani et al., 2017, "Attention Is All You Need"）。

#### 核心组件：

**（1）自注意力机制（Self-Attention）**

自注意力是 Transformer 的灵魂。它让模型在处理每个词时，能够"看到"整个输入序列中的所有词，并根据相关性分配不同的权重。

- 每个 token 会被映射为三个向量：**Query（查询）、Key（键）、Value（值）**
- 通过 Q 和 K 的点积计算任意两个 token 之间的"注意力分数"
- 用注意力分数对 V 加权求和，得到每个位置的输出

公式：$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

**直观理解**：读一句话时，你看到"苹果"这个词，会下意识地关注周围词语来判断它是指水果还是公司。自注意力机制就是在数学上模拟这个过程。

**（2）多头注意力（Multi-Head Attention）**

使用多组不同的 Q、K、V 映射，让模型从不同角度（语义、语法、位置等）同时学习注意力模式，类似于卷积神经网络中的多个卷积核。

**（3）位置编码（Positional Encoding）**

Transformer 本身不具备处理序列顺序的能力（不像 RNN 天然有序），因此需要给每个 token 加上位置信息。常见方法有：
- **正弦位置编码**（原始 Transformer）：用不同频率的正弦/余弦函数编码位置
- **可学习位置编码**：将位置作为可训练参数
- **RoPE（旋转位置编码）**：目前主流方案，通过旋转矩阵编码相对位置，在推理时支持外推到更长的上下文

**（4）前馈神经网络（Feed-Forward Network, FFN）**

每个 Transformer 层中，在注意力之后接一个两层全连接网络，引入非线性变换，增强模型的表达能力。

**（5）残差连接与层归一化**

- **残差连接**：让信息可以跨层直接传递，解决深层网络的梯度消失问题
- **层归一化**：稳定训练过程，加速收敛

#### 三种 Transformer 变体：

| 类型 | 代表模型 | 特点 |
|------|---------|------|
| **Encoder-Only** | BERT | 双向注意力，擅长理解任务（分类、抽取） |
| **Decoder-Only** | GPT 系列 | 单向（因果）注意力，擅长生成任务，目前主流 |
| **Encoder-Decoder** | T5, BART | 同时包含编码器和解码器，擅长序列转换（翻译、摘要） |

现代大模型（GPT、LLaMA、DeepSeek 等）几乎全部采用 **Decoder-Only** 架构，因为其自回归生成能力最强且容易扩展到超大规模。

### 2.2 训练流程

现代大模型的训练通常分为以下阶段：

```
┌─────────────────────────────────────────────────────────────┐
│  阶段一：预训练 (Pre-training)                                │
│  数据：数万亿 token 的互联网文本、书籍、代码                     │
│  任务：预测下一个 token（Next Token Prediction）                │
│  目标：学会语言模式、知识、推理                                  │
│  算力：数千块 GPU，训练数周至数月                                │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  阶段二：监督微调 (Supervised Fine-Tuning, SFT)                │
│  数据：数万至数十万条高质量人工标注的问答/指令数据                │
│  任务：学习遵循人类指令的格式和能力                              │
│  目标：让模型"会聊天"，理解指令格式                              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  阶段三：人类反馈强化学习 (RLHF) / 偏好对齐                      │
│  数据：人类对模型多个输出的偏好排序数据                          │
│  任务：训练奖励模型 → 用 PPO/DPO 优化策略                        │
│  目标：让模型输出"有用、真实、无害"，对齐人类价值观               │
└─────────────────────────────────────────────────────────────┘
```

**补充概念**：

- **DPO（Direct Preference Optimization）**：简化了 RLHF 流程，直接从偏好数据中优化模型，不需要单独训练奖励模型，近年来越来越流行。
- **MoE（Mixture of Experts，混合专家）**：一种模型架构，将 FFN 层拆分为多个"专家"子网络，每次只激活其中一部分（通常 1-2 个）。优势是总参数量大但计算量仅相当于激活参数量的稠密模型，实现了"大容量、低成本"。DeepSeek-V2/V3、Mixtral 等均采用此架构。

### 2.3 扩展法则（Scaling Laws）

OpenAI 在 2020 年提出的 **Scaling Laws** 表明：模型的性能（用测试损失衡量）与**模型参数量、数据量、计算量**之间遵循幂律关系。

这意味着：
- 增大模型、增加数据、投入更多算力都能稳定提升性能
- 三者需要协调增长：大模型需要大数据，数据不够则训练不充分
- **Chinchilla 定律**（DeepMind, 2022）：给定计算预算，最优策略是让 token 数和参数量**同步增长**（约 20 个 token 对应 1 个参数）

| 2018.10 | Google 发布 BERT | 双向理解模型，横扫 11 项 NLP 任务纪录 |

### 3.2 关键里程碑解读

#### ChatGPT 的里程碑意义

ChatGPT 将大模型从"实验室玩具"变成了"大众产品"：
- 首次让普通人体验到大模型的能力
- 引入了 RLHF（人类反馈强化学习），让模型输出更符合人类期望
- 引发了全球范围内的"百模大战"

#### 开源 vs 闭源

- **闭源阵营**：OpenAI（GPT-4）、Anthropic（Claude）、Google（Gemini）
- **开源阵营**：Meta（LLaMA）、DeepSeek、阿里（Qwen）、智谱（GLM）、Mistral

开源模型的快速追赶是大模型发展的重要趋势 —— 当前开源模型已能在多数任务上与闭源模型匹敌。

#### 推理模型的兴起（2024-2025）

以 OpenAI o1 / o3 和 DeepSeek-R1 为代表的**推理模型**引入了"思维链"机制：
- 模型在输出最终答案前，先进行较长的中间推理（"思考"）
- 通过强化学习训练模型的推理过程
- 在数学、编程、科学推理等任务上大幅超越传统大模型

## 四、国外主要大模型介绍

OpenAI 成立于 2015 年（最初为非营利组织，后转为"有限盈利"结构），总部位于美国旧金山，是大模型领域的开创者和持续引领者。

| **多模态** | GPT-4o 支持文本、图像、音频的输入与输出 |
| **核心贡献** | 提出 GPT 系列范式、RLHF 对齐方法、Scaling Laws、推理模型（o1/o3） |

| 版本 | 时间 | 参数 | 里程碑意义 |

**教学要点**：OpenAI 的产品线已分化为两条：**GPT 系列**（快速响应、多模态、通用对话）和 **o 系列**（深度推理、数学/编程/科学），代表了"快思考"与"慢思考"两种范式。

### 4.2 Anthropic — Claude 系列

Anthropic 成立于 2021 年，由前 OpenAI 员工（Dario Amodei 等）创立，总部位于美国旧金山，以 AI 安全和对齐为核心使命。

| **核心创新** | Constitutional AI（宪法 AI）、RLHF 改进、计算机使用（Computer Use） |
| **特点** | 以安全性和有用性著称，编程能力业界领先，长上下文达 200K tokens |

**Claude 系列演进**：

| 版本 | 时间 | 亮点 |
|------|------|------|
| Claude 1 | 2023.03 | 首个产品，主打安全对齐 |

**教学要点**：Anthropic 的核心哲学是"安全第一"，提出的 **Constitutional AI** 方法让模型通过一套"宪法"原则进行自我监督和纠偏，而非完全依赖人类标注。此外，Claude Code（CLI 编程工具）和 Computer Use（让模型操控计算机界面）是 Agent 落地的代表案例。

## 六、补充教学参考

### 6.1 推荐入门论文（按时间顺序）

1. **"Attention Is All You Need"**（Vaswani et al., 2017）— Transformer 架构奠基之作
2. **"BERT: Pre-training of Deep Bidirectional Transformers"**（Devlin et al., 2018）— 双向预训练的里程碑
3. **"Language Models are Few-Shot Learners" (GPT-3)**（Brown et al., 2020）— 大模型时代开启
4. **"Training Compute-Optimal Large Language Models" (Chinchilla)**（Hoffmann et al., 2022）— 训练最优配比
5. **"Training language models to follow instructions with human feedback" (InstructGPT)**（Ouyang et al., 2022）— RLHF 方法论
6. **"LLaMA: Open and Efficient Foundation Language Models"**（Touvron et al., 2023）— 开源大模型的标杆

### 6.2 关键趋势（供课堂讨论）

- **推理模型**：从"快答"走向"深度思考"，思维链成为标配
- **混合专家（MoE）**：在不大幅增加计算量的前提下扩大模型容量
- **长上下文**：从 4K → 128K → 1M，理解完整代码库或长篇论文成为可能
- **多模态融合**：文本 + 图像 + 音频 + 视频的统一理解与生成
- **智能体（Agent）**：大模型 + 工具使用 + 环境交互，自主完成复杂任务
- **端侧部署**：小模型蒸馏与量化，让大模型能力跑在手机和 IoT 设备上
- **AI 安全与对齐**：解决幻觉、偏见、越狱等问题，让模型更可信
